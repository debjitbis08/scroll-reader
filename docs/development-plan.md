# Scroll Reader — Development Plan

## Technology Decisions

| Concern | Choice | Notes |
|---------|--------|-------|
| Package manager | pnpm workspaces | Monorepo — one `pnpm-workspace.yaml` |
| Web framework | Astro 5 (Node.js adapter) | SSR, SolidJS islands |
| UI reactivity | SolidJS | Islands in Astro, full app in Tauri webview |
| CSS | Tailwind CSS v4 | CSS-first config via `@tailwindcss/vite` |
| Theme | Catppuccin (`@catppuccin/tailwindcss`) | Latte (light) / Mocha (dark), auto via `prefers-color-scheme` |
| Icons | Astro Icon + Iconify (`@iconify-json/mdi`) | MDI icon set throughout |
| ORM | Drizzle ORM + drizzle-kit | Schema in `packages/db`, migrations via drizzle-kit |
| Env vars | Astro typed env (`astro:env/server`) | All env vars defined in `astro.config.mjs` `env.schema`, loaded from repo root `.env` via `vite.envDir` |
| Auth | Supabase Auth | Email/password. `@supabase/ssr` for cookie-based SSR sessions. |
| Identity anchor | `profiles` table | `profiles.id = auth.users.id` (via server insert on register) |
| Desktop shell | Tauri (Rust) | SolidJS + Tailwind in webview |
| Local DB | SQLite via Tauri SQL plugin | Inbox state, card cache |
| Encryption | libsodium (`sodiumoxide` crate) | Desktop E2E only |
| AI abstraction (web) | TypeScript interface | Gemini + Ollama behind `AIProvider`. Env: `AI_PROVIDER`, `GEMINI_API_KEY`, `OLLAMA_BASE_URL` |
| AI abstraction (desktop) | Rust trait | All AI calls from Rust — never JS |
| Document parsing | Rust crates | `epub=2` + `scraper=0.21` (EPUB), `lopdf=0.32` (PDF). Binary protocol: JSON stdin/stdout |
| Hosted DB | Supabase Postgres | Direct Drizzle connection via `DATABASE_URL` (transaction pooler, port 6543) |
| Object storage | Supabase Storage (future) | Not yet wired — uploads go to `/tmp` for processing |
| Background processing | `setImmediate` fire-and-forget | `jobs` table tracks state. Batched card generation with progress updates. |

**Self-hosted path skipped.** Focus is hosted web + desktop.

## Phase 2 — Hosted Web App

Goal: users can register, upload a document, and get reading cards back.

### 2.1 Auth ✅ DONE

- [x] `@supabase/supabase-js` + `@supabase/ssr` installed
- [x] `createSupabaseServer(request, cookies)` helper — used in middleware and API routes
- [x] Middleware validates Supabase session → sets `locals.user`
- [x] `/login`, `/register` pages (Catppuccin-themed, form POST)
- [x] `/api/auth/login`, `/api/auth/register`, `/api/auth/logout` API routes
- [x] Register creates `profiles` row via Drizzle after `supabase.auth.signUp`

### 2.2 Upload + Background Pipeline ✅ DONE

- [x] `jobs` table in DB schema (`queued → processing → done/failed`)
- [x] `/upload` page — drag-zone, privacy acknowledgment checkbox
- [x] `POST /api/upload` — validates file type + size + privacy, saves to `/tmp`, inserts doc + job, fires `setImmediate(runPipeline)`
- [x] Batched pipeline (`TRIAL_CHUNK_LIMIT=25`, `BATCH_SIZE=5`):
  - Extracts document (calls extractor binary)
  - Chunks all text elements up to `TRIAL_CHUNK_LIMIT`
  - Inserts all chunks to DB upfront
  - Generates cards in batches of `BATCH_SIZE`, updating `card_count` after each batch
  - Resumable on crash — skips chunks that already have cards
- [x] `/api/jobs/[id]` — authenticated job status endpoint
- [x] `JobPoller.tsx` — SolidJS island that polls every 2.5 s, reloads on completion
- [x] `AutoRefresh.tsx` — SolidJS island that reloads page during generating phase (shows partial cards)
- [x] `/doc/[id]` — document view: chunking spinner → generating progress + partial cards → full card view
- [x] `DELETE /api/documents/[id]` — ownership-verified cascading delete
- [x] Delete button on doc page header with confirm dialog

### 2.3 DB Migrations ✅ DONE

- [x] `drizzle.config.ts` loads `.env` from repo root via `dotenv`
- [x] `pnpm --filter @scroll-reader/db generate` + `migrate` workflow

## Phase 3 — Desktop App (Tauri)

### 3.1 Project Setup

- [ ] `apps/desktop/` — Tauri + SolidJS + Tailwind v4
- [ ] Tauri plugins: `tauri-plugin-sql` (SQLite), `tauri-plugin-keyring`, `tauri-plugin-dialog` (dir picker)
- [ ] Local SQLite schema: `local_files` + `local_card_cache`
- [ ] Settings in SQLite: `server_url` (default hosted), `ollama_url`, `ollama_model`

### 3.2 File Watcher + Inbox

- [ ] `notify` crate watches user-selected directories
- [ ] New EPUB/PDF: hash check against `local_files` — skip if known
- [ ] Unknown files → `inbox_status = 'discovered'`
- [ ] Inbox page: title, author, type, size, language, path
- [ ] Actions: **Add to Library** (approve → process), **Ignore** (permanent hash), **Decide Later**

### 3.3 Local Processing Pipeline (Rust)

- [ ] EPUB + PDF extraction: reuse `packages/extractor` crate (linked in, not subprocess)
- [ ] Chunker: reuse `packages/chunker` crate (linked in)
- [ ] Rust AI provider trait: `trait AIProvider { fn generate(&self, prompt: &str) -> Result<String> }`
  - `GeminiProvider` — API key from OS keychain via `tauri-plugin-keyring`
  - `OllamaProvider` — URL from settings
- [ ] All AI calls via Tauri Rust commands — JS never touches the API key or makes the HTTP call
- [ ] Startup health check for Ollama if selected

### 3.4 E2E Encryption + Sync

- [ ] Keypair derived from password via Argon2id (`sodiumoxide`)
- [ ] Private key in OS keychain, public key synced to `profiles`
- [ ] Card encryption: NaCl `box_` (`cards.front`, `cards.back`, `chunks.content`)
- [ ] Sync encrypted cards to hosted backend (`encrypted: true`)
- [ ] Feed decryption: client-side on load, cached in SolidJS store, cleared on logout
